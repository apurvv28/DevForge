# Contributing to DevForge

Thanks for helping improve DevForge.

## Getting Started

1. Install dependencies with `npm ci`.
2. Run `npm run build` to verify TypeScript compiles.
3. Run `npm test` before opening a pull request.

## Workflow

- Keep commits in Conventional Commits format.
- Add or update tests for behavior changes.
- Avoid introducing new generated files into source control unless they are part of the release docs.

## Release Notes

Releases are automated through semantic-release on `main`.

## Questions

Open an issue or use GitHub Discussions if the repository enables them.
